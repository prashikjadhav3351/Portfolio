Trading Dashboard — Strategy Signal Visualizer

A high-performance, web-based visualization tool designed for quantitative traders and algorithmic developers. It transforms complex, raw strategy data into interactive, human-readable candlestick charts to facilitate signal validation and technical analysis.

Project Overview

The Trading Dashboard provides an intuitive interface for reviewing strategy backtests and monitoring live signals. By processing raw .trade_data files, it visualizes price action combined with technical indicators and automated BUY/SELL signal mapping, allowing traders to quickly identify performance gaps or logic errors.

Technology Stack

Plotly (Py/Js)
Python (Regex/JSON)
Jinja2

System Workflow

1

Data Ingestion

The system monitors and reads .trade_data files containing raw JSON logs of price action and strategy metrics.

2

Sanitization & Parsing

A dedicated regex pre-processor "cleans" non-standard JSON (NaNs, single quotes) before the backend parses the core data.

3

Indicator Calculation

Automatically extracts Moving Averages (LMA, HMA) and OHLC price levels for each ticker from the cleaned dataset.

4

Signal Mapping

Identifies BUY/SELL events by comparing positions and timestamps, mapping them directly onto the price candles.

5

Interactive Rendering

Displays tickers in a Grid View with a Focus Mode for deep inspection of specific trading windows.

Key Features

🎯

Intelligent Signal Markers

Plots "BUY" (Green Up-Triangle) and "SELL" (Red Down-Triangle) markers at precise execution points.

⚡

Trade Execution Lines

Draws horizontal lines at the last trade price with a "soft glow" effect to highlight active entry/exit levels.

📅

Automated Day Division

Vertical dotted lines automatically separate trading days, making multi-day analysis seamless.

🖥️

Clean UI Analytics

Minimalist design with mode-bars removed to focus entirely on price action and indicator relationships.
